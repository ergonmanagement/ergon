---

- Invent new architecture patterns

# 1. Architecture Overview

## 1.1 Stack

Frontend:
- Next.js (App Router)
- TypeScript
- Server Components where appropriate
- Client Components only when necessary

Backend:
- Supabase Edge Functions (primary backend logic layer)

Database:
- Supabase Postgres

Authentication:
- Supabase Auth (email/password only in v1)

Storage:
- Supabase Storage (private buckets)

AI:
- OpenAI API
- LangGraph for orchestration

Billing:
- Stripe

Hosting:
- Vercel (frontend)
- Supabase (DB/Auth/Storage/Functions)

---

## 1.2 Repo Structure Contract (Required)

Cursor must follow this repository structure. Do not invent alternate structures.

### Required Directories

- `/docs`

- `/app/(public)`
- `/app/(app)`
- `/components`
- `/lib`
- `/lib/types`
- `/supabase/migrations`

### Supabase Functions Structure

- `/supabase/functions/onboarding`
- `/supabase/functions/jobs`
- `/supabase/functions/customers`
- `/supabase/functions/schedule`
- `/supabase/functions/finance`
- `/supabase/functions/marketing`
- `/supabase/functions/billing`

### Boundary Rules

- UI components never contain business logic.
- Business logic lives in Edge Functions and shared `/lib` utilities.

- Only Marketing functions may call OpenAI.
- Only Billing functions may call Stripe.

# 3. Multi-Tenancy Model (CRITICAL)

Every table except `companies` must include:

- company_id (uuid NOT NULL)

Every query must filter by company_id.

No exceptions.

---

# 4. Database Schema (STRICT)

All tables include:

- id uuid primary key
- company_id uuid not null
- created_at timestamptz default now()
- updated_at timestamptz default now()
- created_by uuid
- updated_by uuid

# 5. RLS (MANDATORY)

Enable RLS on:

- users
- customers
- jobs
- job_photos
- calendar_events
- finance_entries
- marketing_assets

Policies enforce:

row.company_id = current_user.company_id

Mapping:

auth.uid() → users → company_id

Never disable RLS.

---

# 6. Edge Function Requirements

All Edge Functions must:

1. Validate JWT
2. Extract user id
3. Load company_id
4. Enforce company scoping
5. Validate input schema (Zod recommended)
6. Return structured JSON

---

## 6.1 API Contract Requirement

Every Edge Function must document at the top:

- Request JSON schema
- Query parameters
- Auth requirement
- Response JSON schema (success + error shape)

Use consistent error format:

```json
{
  "error": "message",
  "code": "ERROR_CODE"
}
```

---

## 6.2 Job Photo Upload Flow

1. Client requests signed upload URL via Edge Function.
2. Edge Function validates job ownership (company_id).
3. Edge Function generates signed upload URL.
4. Client uploads directly to Supabase Storage.
5. Edge Function inserts job_photos row.

No direct bucket writes from client without signed URL.
